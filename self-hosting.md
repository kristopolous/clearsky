Clearsky helps users run services themselves by replacing cloud dependencies with containerized, self-hosted alternatives that run entirely on their own machines. Instead of storing photos on Google's servers, users run Immich locally—a self-hosted photo management service with albums, facial recognition, and search, all accessible through a familiar web interface. Clearsky automates the entire setup: it uses Podman to run Immich in an isolated container, configures persistent storage in `~/.clearsky/immich`, and provides import tools that preserve metadata and album structure. Remote access is handled through Tailscale, which creates a secure VPN tunnel so users can access their services from anywhere without exposing their home network. The modular migration framework means each service integration is a self-contained component, making it easy to add new self-hosted alternatives as they emerge.

The key insight is that self-hosting shouldn't require becoming a system administrator. Clearsky hides the complexity—users never see Docker commands, network configuration, or container management. Instead, they interact with a simple GUI wizard that guides them through exporting data from cloud services and importing it into their self-hosted setup. If something goes wrong, the rollback feature atomically reverts to the previous state, giving users confidence to try self-hosting without fear of breaking their system. By bundling everything into a single AppImage built with Nix, Clearsky ensures the self-hosted setup works identically across different Linux distributions, removing another barrier to entry. This transforms self-hosting from a technical challenge into a straightforward migration process, making privacy and data sovereignty feel like normal, everyday choices rather than advanced technical feats.
